# Security Policy

Thank you for helping keep Handle and its users safe.

## Supported versions

At this time, security fixes are provided on a best-effort basis for:
- the `main` branch
- the latest tagged release, if applicable

Older releases may not receive security updates.

## Reporting a vulnerability

Please **do not** report security vulnerabilities in public GitHub issues, pull requests, or discussions.

Instead, report them privately to:

support@tonkotsu.ai

Please include, if possible:
- a clear description of the issue
- affected version, tag, or commit
- reproduction steps or proof of concept
- impact assessment
- any suggested mitigation

## What to expect

We will aim to:
- acknowledge receipt within a reasonable timeframe
- investigate and validate the report
- coordinate on a fix and disclosure timing if the report is valid

Please do not publicly disclose the issue until we have had a reasonable opportunity to investigate and address it.

## Scope

Examples of issues that should be reported here:
- remote code execution
- privilege escalation
- auth or access-control bypass
- secret leakage caused by the software
- code injection
- unsafe defaults that create real security risk

Examples of issues that usually do **not** belong here:
- requests for general hardening advice
- theoretical concerns without a plausible exploit path
- issues in third-party services outside the control of this project
- support questions or configuration help

For general help, see `SUPPORT.md`.
