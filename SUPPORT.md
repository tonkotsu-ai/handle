# Support

Thanks for using Handle.

## Where to get help

### 1. Read the documentation first

Please start with:
- the `README.md`
- any docs in this repository
- existing issues and pull requests

## 2. For bugs

If you believe you found a reproducible bug, please open a GitHub issue using the bug report template.

Please include:
- what you were trying to do
- what you expected to happen
- what actually happened
- steps to reproduce
- environment details
- logs or screenshots if helpful

## 3. For feature requests or proposed changes

Please open an issue using the feature request template **before** opening a pull request.

Handle is maintained with a **maintainer-led roadmap**. We welcome ideas, but not all proposals will be accepted. Discussing changes first helps confirm whether a contribution is likely to be a fit.

## 4. For security issues

Do **not** file public issues for security vulnerabilities.

Report them privately to:

support@tonkotsu.ai

See `SECURITY.md` for details.

## 5. What support to expect

This project is maintained on a best-effort basis. We may not be able to respond to every request, feature proposal, or support question.

Please be patient, and keep reports focused and actionable.

## 6. What not to use GitHub issues for

Please avoid opening GitHub issues for:
- general troubleshooting without enough detail to reproduce
- feature work that has not been discussed first
- security disclosures
- off-topic questions unrelated to the project
